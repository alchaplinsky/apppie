Grids

Grids system is a way to break any space into fractions, to make layout on a page cohesive. Grids like a lattice divides horizontal space in consistent columns where text, images, elements can be placed. Vertical space is controlled by height of content.

A lot of different grid frameworks are created, which fit different needs, but this grid system has certain benefits:

  • grid is defined in one place
  • it is reusable
  • it is efficient
  • simple class naming
  • grids are adaptive, we don't use fixed width
  • no predefined spaces
  • grids are nested
  • consistent behaviour across all common browsers

Grid is defined in one place

All grids styles gathered in one small document grids.css in folder "base" including other css files with predefined styles. That reduces the amount of CSS and markup you need to duplicate each time you start a new project.

Grid is efficient

Grids system is built using columns, where column is the smallest unit of measurement. Predefined to element's class property float makes the element simply fall into place on the page. All elements are wrapped in div with class .row. The parent element will collapse to zero height since it has only floated children. It should be fixed by clearing it.

.row:after {
  content: ".";
  display: block;
  height: 0;
  line-height: 0;
  clear: both;
  visibility: hidden; 
}

Relative sized grids can be created by dividing an element into halves, thirds, fourths, or fifths. Maximum 6 columns grid is used when design layout. Column width is assigned by CSS class.

Simple class naming

Simple name classes are used in grids. Class names indicates the fractional width of the column in %, for example .col1of3 would take up one quarter, or 33.33333%, of the horizontal space. The following fractions are supported: col1of2, .col1of3, .col2of3, .col1of4, .col2of4, .col3of4, .col1of5, .col2of5, .col3of5, .col4of5, .col1of6, .col2of6, .col3of6, .col4of6, .col5of6.

Classnames shouldn't have any additional info about grids, like align or spacings.

Grids don't use fixed width'

Layout should be abstracted and not contain elements with fixed width. Wrapped elements of higher level should define widths of inside components. So width of columns can be defined using % of parent div.

.col1of2, .col2of4, .col3of6 {
  width: 50%; 
}

.col1of3, .col2of6 {
  width: 33.33333%; 
}

.col2of3, .col4of6 {
  width: 66.66666%; 
}

.col1of4 {
  width: 25%; 
}

.col3of4 {
  width: 75%; 
}

.col1of5 {
  width: 20%; 
}

.col2of5 {
  width: 40%; 
}

.col3of5 {
 width: 60%; 
}

.col4of5 {
  width: 80%; 
}

.col1of6 {
  width: 16.66666%; 
}

.col5of6 {
  width: 83.33333%; 
}

No predefined spaces

Property box-sizing is used so all spaces inside grid were optional. When we set a width to element, that element stays that width, despite padding or borders being applied.

.col1of2,
.col1of3, .col2of3,
.col1of4, .col2of4, .col3of4,
.col1of5, .col2of5, .col3of5, .col4of5,
.col1of6, .col2of6, .col3of6, .col4of6, .col5of6 {
  float: left;
  -webkit-box-sizing: border-box;
  -moz-box-sizing: border-box;
  box-sizing: border-box; 
}

Grids are nested

Complicated layouts can be built without creating a lot of custom elements.. A column can contain another .row with a lot of columns inside or it can contain other objects directly. How deep and complicated are grids depents on designs, but there is no need to create new classes. The sum of the fractional sizes of all children of one .row should be equal to one. Grids control width, content controls height.
