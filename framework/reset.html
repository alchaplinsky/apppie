Reset.css

The goal of a reset stylesheet is to reduce browser inconsistencies in things like default line heights, margins and font sizes of headings, and so on. CSS reset is the first stylesheet that should be presented to browser, so it makes browser to play by our rules.

There are multiple variations of reset.css and normalize.css al over the web, icluding most popular Minimalistic Reset (setting margins and paddings of all elements to 0), Eric Mayer's reset and Yahoo reset. But most of them are just a starting point, and they should be extended, edited, tweaked to match your specific reset baseline. Reset presented below is a mix of different reset techniques that solve different issues and inconsistencies in browsers behaviour, and it's complete enough to include it to a project and extending step.

Block 1: 
* {
  padding: 0;
  margin: 0;
  outline: none;
  font-size: 1em;
}

Explanation 1: These lines reset padding and margin for all elements on a page. Also it kills the default outline style and sets font-size to 1em, so elements like sup, sub, small and h1..h6 will be the same size as a regular text. all font properties for them need to be defined in typography file.

Block 2: 
img, iframe, fieldset, object, table {
  border: none;
}

Explanation 2: Here we are killing border for elements that have border by default. Image will appear bordered on a page if it is nested insite a link.

Block 3: 
sub, sup {
  vertical-align: baseline;
}

Explanation 3: As soon as we set font size of 1em for sub and sup elements, we should also normalize it's vertical alignment.

Block 4: 
caption, th {
  font-weight: normal;
  text-align: left;
}

Explanation 4: By default table headers thead > th and caption are bold and center aligned. Getting rid of this with code above.

Block 5: 
table {
  border-collapse: collapse;
  border-spacing: 0;
}

Explanation 5: Removing this 'double border' default effect for tables.

Block 6: 
td {
  vertical-align: top;
}

Explanation 6: Killing default middle alignment of table cell's content.

Block 7: 
th, h1, h2, h3, h4, h5, h6 {
  font-weight: normal;
}

Explanation 7: Setting font weight normal for all headers (they are bold by default) so that we can specify all header styles in typography.

Block 8: 
img {
  -ms-interpolation-mode: bicubic;
}

Explanation 8: If you will use responsive images technique by setting max-width: 100% to images, this will help you with better image rendering in IE.

Block 9: 
button, input, select, textarea {
  margin: 0;
  font-size: 100%;
}

button, input {
  *overflow: visible;
  line-height: normal;
}

button::-moz-focus-inner, input::-moz-focus-inner {
  border: 0;
  padding: 0;
}

Explanation 9: A set of fixes for form elements wich have different behaviour and look in different browsers. First let's ensure that we get rid of default margins for inputs and selects and setting it's font size to 100% which is the same as regular text. After that we'll handle button and input line-height. And finally remove annoying inner padding that is present in firefox.

Block 10: 
article, aside, details, figcaption, figure, footer, header, hgroup, nav, section {
  display: block;
}

Explanation 10: HTML5 tags block appearance on the page.
