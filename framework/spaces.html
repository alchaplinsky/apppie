Spacings

Spacings - is pretty simple concept that can be used in absolutely every project. It allows you to get rid of multiple margin and padding rules in your css code.

If you find yorself writing margin-bottom: 20px for about dozen of times - you might need helper class for it. This means that you have pretty standard bottom spacing (margin) for UI objects, and that's good! That's consistency in design but, that is duplication in code. The same with inner paddings of different blocks and left or right spacings between buttons - all this should be consistent. And we can use helper classes for this, so we don't need to rewrite all this rules in CSS code for hundreds of times.

Let's define 3 different space sizes, let's say small: 5px, medium: 15px and large: 30px and create helper classes for them. Naming is pretty simple and inpuitive: Each class name will consist of three letters, first one defines if it is margin or padding(m or p), second will specify direction top, bottom, right or left (t, b, r, l), and third one tells us which size to use small, medium or large (s, m, l). And also we can define additional direction letters for horizontal (both left and rigt), vertical (both top and bottom) and all directions (h, v, a).

Example:
.pbm - means padding bottom medium == padding-bottom: 15px
.mbl - means margin bottom large == margin-bottom: 30px
.mam - means margin all medium == margin: 15px
.pts - means padding top small == padding-top: 5px

Defining 3 standard spacings for entire UI helps you stay consistent from page to page and eliminates CSS code duplication.

Warning!
But pay attention, you should not abuse spaces helpers! Your code should not look like this: 

<div class="pam mhl mts mbm"></div>

If you have more than 2 spacing helpers on single html tag, that's obvious signal that you need separate classname for this element, where you'll define all this paddings and margins. Helper classes should actually help you build web interfaces, but not make your code hard to read!
