Structure and appearance

You would never buy a car which does not allow you to change tyres. Or appartment where you cannot change the wallpaper.

So the point is, structure should be not mixed with appearance.

Lets outline some bullet points:

  • all appearance attributes are prohibited (height, width, bgcolor, color etc.)
  • inline styles are crime against humanity
  • use inline images ONLY for content pictures
  • avoid elements that do not carry any content (decorating only)
  • use html tags semantically driven: i.e. tables are for table data only

There is a good way to check if your structure is separated. Turn all your CSS files off and look at your page.
If it looks like normally structured page with logical flow and without sudden background images or extra elements then you are good!
