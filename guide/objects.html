Objects

Object Oriented CSS is all about learning to think about your UI and CSS in terms of objects. Object is a simple independent and indivisible component that is used across the project. Objects are small little chunks of functionality. You can think of them as interface elements like headings, form fields, buttons, and content areas.

Object names should be semantic simple and descriptive.

Object's independence means that no external styles or styles of other objects shoul influence to object's look or behaviour. It should look the same if it is placed on empty page as a single element either if it is placed in context within complex layout with multiple other objects.

Object's indivisibility means that object is a smallest structural unit of interface and there is a small chance to take smaller parts from it to a separate object. For instance if we take a look at navigation with following structure:

<ul class="navigation">
  <li><a href="/">Home</a></li>
  <li><a href="/about">About</a></li>
  <li><a href="/login">Login</a></li>
</ul>

It's not likely that we would need to divide this object to elements, because we won't use list item li without actual list element ul. But we can take out a link element. But link should have it's default styles already defined in typography.
