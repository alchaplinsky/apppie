Categorizing CSS Rules

Every project needs some organization. Throwing every new style you create onto the end of a single file would make finding things more difficult and would be very confusing for anybody else working on the project. It is important to have a standard high-level structure in place. proposed structure

There are four types of categories:

  • Reset
  • Toolkit Styles (grids, helpers, etc.)
  • Typography
  • Layout
  • Objects
  • Modules
  • Sections

Base rules are the defaults. Reset.css, grids, spaces and util classes, that stays the same across all the projects.

Layout rules divide the page into sections. Layouts hold one or more modules together.

Typography rules define basic text styling. All types of headers that are used in project, text and link styles, fonts and colors are defined here.

Modules are the reusable, modular parts of our design. They are the ui elements, buttons, inputs, lists, blocks and more complex modules.
