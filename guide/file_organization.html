File Organization

In general, the CSS file organization in rails app should follow something like this:

stylesheets
  ├── base
  │   ├── mixins.sass
  │   ├── reset.sass
  │   ├── grids.sass
  │   ├── spaces.sass
  │   └── helpers.scss
  ├── application.sass
  ├── application
  │   ├── layout.sass
  │   ├── typography.sass
  │   ├── modules
  │   |   ├── buttons.sass
  │   |   ├── forms.sass
  |   |   └── ..........
  │   ├── sections
  │   |   ├── profile.sass
  │   |   ├── dashboard.sass
  |   |   └── ..........
  │   └── responsive
  │       ├── tablet.sass
  │       ├── mobile.sass
  |       └── ..........
  └── shared
      ├── popup.scss
      ├── markdown.scss
      └── ..........

base folder contains almost unchangeable base styles for the project. These are mixins (if you use sass), reset, grids, spaces, fonts (for including custom fonts) and utils (util css classes for text align and positioning)

application.css manifest file which defines what files should be included. Manifest is that it has the same name as rails layout file app/views/layouts/application.html.erb. It is likely you'll need to create more manifests for each rails layout you have in the app, because they might look different and have different styles.

application folder is named after sass manifest, and contains all the styles (except base, shared and responsive) that are used in application layout. It includes layout styles, modules, sections, typography etc.

shared contains styles that need to be shared between different layouts. For instance you have 2 layouts/application.html.erb and layouts/landing.html.erb, so you should create application and landing folders under stylesheets directory as well as application.sass and landing.sass manifests. And if you use the same styles for markdown formatting in both of them, you should place them into shared folder.

application/layout.sass Here we define layout structure and representation.

application/typography.sass Here is the file where custom fonts are included and all text styles are defined. Font family and fonts sizes for paragraphs and headings, link styles and behaviour etc.

application/modules All UI objects go here. Each object or group of similar objects in a separate file.

application/sections Place for all structures, that aren't basicaly objects, but define some blocks or structures containing objects.

application/responsive Folder for all the responsive styles (media queries should go).
